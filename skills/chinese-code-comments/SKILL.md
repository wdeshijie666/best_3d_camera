# 中文代码注释（camera_3d_stack）

## 原则（与 `code-comments-skill.txt` 一致）

1. **先读逻辑再下笔**：结合当前函数/类职责与调用关系写注释，不写「代码在做什么」的复述型废话。
2. **语言**：注释主体为**中文**；API/协议名、类型名、与外部文档一致的英文术语保留英文。
3. **风格**：参考 **Google C++ Style Guide** 的注释分工——文件/类/公开 API 用简短块注释说明意图与不变量；复杂分支、单位、时序、线程假设用行内或局部块注释点破。
4. **精简**：默认一两句说清；**仅在不写会误读、踩坑、或违反协议时**再展开（例如 SHM 代际、gRPC 错误码与业务码映射）。
5. **禁止**：与当前逻辑无关的科普、复制粘贴的需求文档、已能从标识符看出的信息、大段被注释掉的死代码（除非用户明确要求保留并标注原因）。

## 落地动作

- **新增代码**：公共头/模块入口在文件头或类上给出 **@file / 职责** 级一句说明（与根 `README.md` §6 一致时可引用「见 code-comments-skill.txt」不必重复全文）。
- **修改代码**：注释与实现同步；删逻辑则删误导性旧注释。
- **Qt**：信号槽、跨线程 `Qt::QueuedConnection` 等易错点优先一句点明。

## 自检

- 删掉注释后，维护者是否仍能从命名+结构理解？能则考虑再压缩注释。
- 是否存在「英译中式的空话」？有则删掉或改成具体约束/原因。
